First time installing? Read this.

Omnimorf is a brand-new independent app, so Windows and macOS will show a security warning the first time you run it. This is not a virus warning — it just means the OS doesn't recognize the developer signature yet. Here's exactly what you'll see and how to continue.

⊞ Windows — "Windows protected your PC"
  1. Double-click Omnimorf-Setup-Windows.exe
  2. A blue box titled "Windows protected your PC" will appear
  3. Click the small "More info" link in the top left
  4. A new "Run anyway" button will appear — click it
  5. The normal installer opens. Follow the wizard.

Why it happens: Microsoft SmartScreen flags all new apps until they build up enough download reputation. The warning disappears automatically after enough users install it. Nothing in Omnimorf changes — it's the exact same file, just no longer flagged.

macOS — "Omnimorf can't be opened"
  1. Open the downloaded .dmg and drag Omnimorf to Applications
  2. Open Finder → Applications → right-click (or Control-click) on Omnimorf
  3. Choose Open from the menu
  4. A dialog appears with an "Open" button — click it
  5. Omnimorf launches. You only have to do this once.

If you still see "Omnimorf is damaged and can't be opened", open Terminal and run:
xattr -cr /Applications/Omnimorf.app
Then try opening it normally. This clears macOS's quarantine flag on the downloaded file.

🐧 Linux — AppImage
  1. Download Omnimorf-Linux.AppImage
  2. Right-click → Properties → Permissions → check "Allow executing file as program" (or run chmod +x Omnimorf-Linux.AppImage)
  3. Double-click to run. No installation needed.

How to verify this isn't malware: Every build is compiled in the open on GitHub Actions from source code you can inspect. Check the SHA-256 of your downloaded file against the one published on the Releases page — if they match, the file is byte-identical to what our CI produced. Omnimorf makes zero network calls during file conversion — your files never leave your machine.
